waitress

先前我們已經知道可以利用 uwsgi 在 Ubuntu 平台配置 CMSiMDE 動態網站. 至於在 Windows 希望採 Production 方式部署 Flask 網際程式, 則可以使用 waitress. 因為在 Windows 上若希望使用 uwsgi, 只能透過老舊的 Cygwin.

因為在近端啟動 CMSiMDE 動態網站, 可以透過 python cmsimde/wsgi.py 執行. 若要在 cmsimde 目錄中使用 waitress, 則需要先以 pip install waitress 安裝模組, 之後另外建一個啟動檔案 waitress_server.py:

# pip install waitress
# this is cmsimde/waitress_server.py
from waitress import serve

# under cmsimde directory import flaskapp
import flaskapp

# run cmsimde dynamic site with production waitress
serve(flaskapp.app, host='0.0.0.0', port=8080)

並且採用 python cmsimde/waitress_server.py 啟動. (若在近端啟動 CMSiMDE 動態網站進行多人協同, 也可以透過 waitress.)

接下來應該可以利用上述的方式部署 https://github.com/mdecourse/fossilapp 與 https://github.com/mdecourse/fossiloauth 等兩個應用, 設法與前面的 Fossil SCM 結合啟用. 所能得到的結果是, 使用者可以利用 Gmail 帳號登入 Fossil SCM 網際伺服器參與 Forum 討論, 或者登入後自行在多 repository 的 Fossil SCM 網際伺服器上建立所需要的倉儲. 當然也可以將 CMSiMDE 靜態網站的資料同步到 Windows 平台上的 Fossil SCM 網際伺服器.
