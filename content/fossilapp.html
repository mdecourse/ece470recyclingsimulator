fossilapp

在 Ubuntu 中設定 Stunnel 作為 http 網際服務的 https reverse proxy 時, 必須將 /etc/environment 中的 HTTPS 設為 on.

而在 Windows 同樣以 Stunnel 作為 fossilapp 應用程式的 reverse proxy 時, 假如以 waitress 啟動 Flask 程式, 則可以利用 waitress 中的 url_scheme='https' 設定, 達到相同目的.

在 Windows 操作系統配置 fossilapp, 所需要的 waitress 伺服程式如下:

# pip install waitress
# this is waitress_server.py
from waitress import serve

# under cmsimde import fossilapp
import fossilapp

# run cmsimde dynamic site with production waitress
serve(fossilapp.app, host='0.0.0.0', port=5000, url_scheme='https')

表示與 fossilapp.py 在相同目錄的 waitress_server.py, 採 waitress production 模式啟動. 透過 http 在 port 5000 啟動, 並且由 reverse proxy https 的協定接受外部 requests. 至於聯外的 port 必須在 Stunnel 中設定.

接著配合的 Stunnel reverse proxy 設定如下:

[https]
accept = 8443
connect = 5000
cert = localhost.crt
key = localhost.key
TIMEOUTclose = 0

而在 https://console.developers.google.com 中的 Oauth 2.0 client 設定則為:

Authorized JavaScript origins
For use with requests from a browser
URIs *		
https://server_domain_name:8443

Authorized redirect URIs
For use with requests from a web server
URIs *		
https://server_domain_name:8443/login/google/

利用 nssm 建立能夠執行 python waitress_server.py 的 Windows 服務, 可以利用 nssm install "fossilapp" 配置名稱為 fossilapp 的系統服務. 該服務的執行主體為 C:\compile_copsim_for_cad2021\data\python395\python.exe, 執行所在目錄必須選擇 Ｃ:\compile_copsim_for_cad2021\data\tmp\fossil\fossilapp, 也就是 fossilapp.py 所在目錄, 且給定執行參數 C:\compile_copsim_for_cad2021\data\tmp\fossil\fossilapp\waitress_server.py.

另外, 為了讓 nssm 所建立的 service 可以正常啟動, fossilapp.py 中讀取 secret file 的路徑也必須使用絕對路徑, 或者必須將該目錄列入 Python 程式執行搜尋目錄, 例如:

with open(r'C:\compile_copsim_for_cad2021\data\tmp\fossil\scrum1_client_secret.txt', 'r') as f:
    key = f.read().splitlines()

以下則為 Windows 環境下 fossilapp 程式中的 login 函式:

@app.route('/login/<provider_name>/', methods=['GET', 'POST'])
def login(provider_name):
    
    # We need response object for the WerkzeugAdapter.
    response = make_response()
    
    # Log the user in, pass it the adapter and the provider name.
    result = authomatic.login(WerkzeugAdapter(request, response), provider_name)
    
    # If there is no LoginResult object, the login procedure is still pending.
    if result:
        if result.user:
            # We need to update the user to get more info.
            result.user.update()

        # only get the string before @
        loginUser = result.user.email.split("@")[0]
        loginDomain = result.user.email.split("@")[1]

        # only @gm.nfu.edu.tw can login
        if loginDomain == "gm.nfu.edu.tw":
            session["user"] = loginUser
            session["login"] = True
            template_lookup = TemplateLookup(directories=[template_root_dir])
            loginTemplate = template_lookup.get_template("login.html")
    
            return loginTemplate.render(result=result, CALLBACK_URL=CALLBACK_URL)
        else:
            return "Sorry, you are not allowed to login."

    # Don't forget to return the response.
    return response
