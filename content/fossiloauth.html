fossiloauth

fossiloauth 能夠利用 Gmail 的 Oauth2 架構登入, 主要根據 https://www.fossil-scm.org/home/doc/trunk/www/password.wiki 中的說明, 當使用者的密碼欄位資料並非 40 字元的 SHA1 hash, 在認證流程中該密碼就會被視為明碼. 因此當 Flask 網際程式利用 Oauth2 讓使用者登入後, 可將使用者的登入帳號與電子郵箱存入 sessions, 之後再利用 SQLite 語法以使用者帳號進入 Fossil SCM 倉儲查核, 若倉儲中已經有該帳號, 則以亂碼更新帳號密碼, 若倉儲中尚無該帳號, 則以登入帳號建立 user, 且以亂碼作為密碼. 接著再根據程式中所暫存的用戶帳號與密碼, 以 ajax post 方式登入目標倉儲後, 將登入後的 sessions 轉到網際 Fossil SCM 伺服器, 開啟 Forum 連結. 這時所顯示的 Forum 狀態即顯示該用戶已經登入. 可以在 Forum 模組中發言.

fossiloauth 的部署與 fossilapp 部署類似, 但 fossiloauth 編寫目的是要讓使用者透過 @gm 帳號登入後, 可以直接用學號參與 Fossil SCM forum 的討論.

由於 fossiloauth 程式利用 session 在 Fossil SCM 與 Flask 程式交換帳號登入內容, 因此在 Ubuntu 主機中需要利用 https://certbot.eff.org/lets-encrypt/ubuntufocal-nginx 進行主機數位簽章認證.

在 Ubuntu 中採 nginx 以 certbot 擷取認證檔案 fullchain4.pem 與 privkey4.pem, 也就是一般 stunnel 設定檔案中的 localhost.crt 與 localhost.key. 而在 Windows 環境中, 主機也應該要透過 https://certbot.eff.org/lets-encrypt/windows-nginx 取得數位簽章認證檔案.

Windows 安裝 nginx

從 https://nginx.org/en/download.html 下載 nginx-1.21.3.zip 並啟動 nginx.

從 https://dl.eff.org/certbot-beta-installer-win32.exe 下載 certbot 並安裝.

目前在 Windows 採用 Stunnel 結合 Flask  fossiloauth, NSSM (source codes: https://github.com/kirillkovalenko/nssm) 與 Fossil SCM 已經可以完成採 Gmail 登入 Fossil SCM 論壇的網際主機.

fossiloauth login.html 最後登出 Gmail 帳號並 redirect 至 fossiloauth forum 函式的設定:

<script type="text/javascript">
window.location="https://www.google.com/accounts/Logout?continue=https://appengine.google.com/_ah/logout?continue=https://${site[0]}:${site[1]}/forum";
</script>
